# LLM 上下文工程实验室：F 函数专家全景图 v6.0

这是一套专为大语言模型（LLM）开发者设计的**上下文管理（Context Engineering）**可视化探索实验室。本项目通过“F 函数”这一数学抽象模型，深度剖析了 14+ 种主流的上下文处理策略，帮助开发者权衡智能忠实度、计算成本与工程复杂度。

## 🧪 核心理念：何为 F 函数？

在 LLM 应用中，上下文演进可以用一个简洁的公式表达：
$$C_{t+1} = F(I_t, O_t, C_{t}, M)$$

其中：
- $I_t$: 当前输入
- $O_t$: 模型输出
- $C_t$: 现有上下文
- $M$: 外部记忆库
- **$F$**: 上下文管理函数 —— 本实验室研究的核心，决定了有限的窗口里该留下什么，舍弃什么。

## 🚀 核心功能

- **14+ 策略深度拆解**：涵盖从基础的“滑动窗口”到硬核的“语义剪枝（LLMLingua）”、“知识图谱索引（KG Index）”等。
- **能力特性指纹图**：多维度（忠实度、Token 效能、工程复杂度、扩展性、逻辑结构化）评估每种策略。
- **数学范式还原**：针对每种方案提供严谨的逻辑表达公式。
- **✨ AI 智能实验室**：内置 Gemini 服务接入，直接针对您的应用场景生成 Python 伪代码实现或避坑指南。

## 📦 策略矩阵

### 1. 基础范式 (Basic)
- **全量缓冲 (Full Buffer)**：100% 还原，极其昂贵。
- **滑动窗口 (Sliding Window)**：FIFO 机制，最常用的兜底方案。

### 2. 压缩型 (Compress)
- **滚动摘要 (Rolling Summary)**：语义蒸发换取超长期记忆。
- **混合缓冲摘要 (Summary Buffer)**：生产级推荐，性能与精度的平衡。
- **Map-Reduce 压缩**：处理百万级 Token 的离线预处理核武器。
- **语义剪枝 (LLMLingua)**：Token 级高倍率脱水。
- **语义权重压缩**：专为任务导向设计的“冷酷”过滤。

### 3. 架构型 (Architecture)
- **向量检索记忆 (Vector RAG)**：语义召回，海量数据的解药。
- **外部文件卸载 (File Offloading)**：模仿现代操作系统的“虚拟内存”思想。
- **分层记忆架构 (Hierarchical Memory)**：对标冯·诺依曼架构的 L1/L2/L3 层次化管理。
- **上下文范围限制 (Scoped Memory)**：多 Agent 协作中的信息隔离技术。

### 4. 结构化 (Structured)
- **言语强化 (Reflexion)**：基于错误的自我进化记忆。
- **事实提取 (Fact Extraction)**：转化为结构化 JSON，数据确定性之巅。
- **知识图谱索引 (KG Index)**：超越语义搜索的逻辑联想推理。
